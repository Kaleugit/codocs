# Motor híbrido: análise estática + LLM

A geração de documentação poderia ser feita com o LLM lendo o código diretamente (agnóstico de linguagem, mas caro e sujeito a alucinar arquitetura) ou apenas com análise estática (precisa, mas incapaz de produzir narrativa). Decidimos o híbrido: a análise estática (tree-sitter + histórico Git) produz o Esqueleto — fatos verificáveis sobre estrutura, dependências, entry points e hotspots — e o LLM escreve apenas a narrativa (ELI5, fluxos semânticos, prosa) em cima desse esqueleto. Consequência: diagramas estruturais são renderizados deterministicamente (Graphviz) a partir do esqueleto e nunca passam pelo LLM; apenas fluxos semânticos são gerados via Mermaid e validados sintaticamente antes da renderização. Critério de qualidade derivado: tolerância zero a fatos inventados; omissão é aceitável, invenção não.
